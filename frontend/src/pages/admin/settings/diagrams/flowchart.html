Key Insight — Standard Order
Supports 8 order types: standard, bulk, b2b, quotation, service, mixed, project, subscription. Customer can only cancel/trash pending+unpaid orders. Admin can cancel any status with full refund logic including returnless refunds and proportional per-item refund distribution.

Key Insight — Hamper Orders
Hamper orders are financially self-contained. The conversion to a standard order copies all financials as-is. After conversion, hamper order status changes are blocked — manage via the standard order instead. Bundle discount is distributed proportionally across items during conversion.

Key Insight — Auction Orders
Auctions use proxy bidding — the system auto-bids up to the user's max. Last 2-minute bids trigger anti-sniping extensions. Admin approves bids with optional per-bid charged_amount overrides. Orders support full refund-on-cancel with stock restoration and payment reversal.

Key Insight — Inventory Architecture
The products table owns stock_quantity. The inventory system mirrors this via available_qty. The issued_qty is a running counter of net lifetime sales — increases on sale, decreases on return. This design maintains a full audit trail through InventoryLifecycleMovement while keeping the products table as the single source of truth.
